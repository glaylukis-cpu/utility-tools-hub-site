完了: v0.4.2 PDF preview research spike

Rust-side rendering、PDFium、pdf.js、WebView embed、OS viewer、lightweight pseudo-previewを比較し、Preview planned文言とQA checklistを整理しました。次の候補はlightweight operation-plan visualization、選択ページ範囲・Merge order・Protected indicatorの可視化です。Inspect PDF summary、Merge multi-file summary、merge order、Total pages、protected PDF warning、既存のMerge / Split / Extract / Rotate / Deleteは引き続き利用できます。PDF preview、rendering、thumbnails、reorderは未実装で、PDF.jsまたはPDFiumによる実験的描画は後続候補です。

v0.4.0

PDF Workbench foundation

PDF Workbench UI shellを整え、PDF inspect core / bridge、ファイルsummary、単一PDF操作の入力summary、Mergeの複数ファイルsummaryを追加しました。選択順、ファイル名、サイズ、ページ数、PDF version、合計ページ数、Normal / Protected status、保護PDFの警告を確認できます。Merge / Split / Extract / Rotate / Deleteは引き続き利用可能で、PDF処理はローカル実行されます。

v0.3.4

Protected PDF merge error handling

PDF Mergeの安定性と実PDFテストを強化し、PDF version固定による互換性リスクを改善しました。暗号化・権限制限付きPDFは未対応として明確に拒否し、通常の未保護PDF同士のMergeは引き続き利用できます。復号・制限回避・パスワード処理や新しいPDF機能は追加していません。

v0.3.3

PDF Tools QA and UX polish

既存のMerge / Split / Extract / Rotate / Deleteについて、操作説明と入力例、disabled理由、loading・success・error表示を改善しました。Delete pagesの安全注意、Planned / Research表示、実PDF確認用QA checklistも整理しています。新しいPDF処理機能やRust PDF core / bridgeの変更はありません。

v0.3.2

PDF rotate / delete MVP

Rust PDF rotate / delete coreとcommand bridgeをPDF Toolsへ接続しました。指定ページの90・180・270度回転、ページ単位の削除、loading・success・error表示を利用できます。ページ削除は墨消しではありません。

v0.3.1

PDF split / extract MVP

Rust PDF split / extract coreとcommand bridgeをPDF Toolsへ接続しました。PDF分割、指定ページ抽出、ページ指定入力、loading・success・error表示を利用できます。Merge PDFsも引き続き利用可能です。

v0.3.0

PDF merge MVP

Rust PDF merge coreとcommand bridgeをPDF Toolsへ接続しました。複数PDF選択、出力先指定、Merge実行、loading・success・error表示を利用できます。
